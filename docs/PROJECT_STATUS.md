## v1.2.7 Changes — 2026-07-02

### Skeet Shooter Complete Overhaul

**Round system with 1-second launch delay:**
- Each round shows target words in bottom bar, then waits 1s before launching skeets
- "Get ready…" hint shown during delay; no skeets move during this window
- Rounds 1-2: 1 target word; Rounds 3-10: 2-3 targets; Rounds 11-20: 3-4; Round 21+: 3-5
- Round completes only when ALL target words are shot; remaining decoys cleared

**Multiple targets per round:**
- `_targetWords` list tracks all words for current round
- `_foundWords` list tracks which have been hit; shown with ✅ and strikethrough in bottom bar
- Correct skeet for each missed word respawns independently after 800ms
- Decoys cleared and new round starts once all targets found

**All skeets identical dark navy (no visual cheating):**
- All skeets: dark navy radial gradient `#2D2A6E → #1a1740` with gold border `#D4A017`
- Gold glow box-shadow; white text with dual black shadow for readability
- Font scales: ≤6 chars → 13pt, ≤10 → 11pt, ≤14 → 10pt, else 9pt; skeet diameter 58px

**Thai Script / Phonetic toggle:**
- New toggle on Skeet Shooter start screen (Learning Thai mode only): `🇹🇭 Thai Script` / `🔤 Phonetic`
- Pill toggle `ก` / `abc` visible in top bar mid-game; switches instantly
- Saved to `SharedPreferences` key `skeet_use_phonetic` via `SettingsService.skeetUsePhonetic`
- Also available in Settings → Arcade section (learningThai mode only)

**Language mode fixed:**
- Learning Thai (EN speaker): bottom bar shows English; skeets show Thai or phonetic
- Learning English (TH speaker): bottom bar shows Thai + phonetic; skeets show English

**8 trajectory patterns (unlock by level):**
- Lv 1+: `leftToRight`, `rightToLeft`
- Lv 5+: `topLeftToBottomRight`, `topRightToBottomLeft`
- Lv 10+: `bottomLeftToTopRight`, `bottomRightToTopLeft`
- Lv 20+: `topToBottom`; Lv 30+: `zigzag` (sine wave)
- Diagonal patterns use perpendicular arc offset for smooth curvature

**Anti-overlap spawning:**
- Up to 5 attempts to find a spawn position ≥80px from all live skeets
- Decoys stagger-spawned 500ms apart per initial batch; respawn after 1-2s random delay

**Randomised speed:** `baseSpeed(lv) × randomFactor(0.7-1.3)`, clamped 80-380 px/s

**Explosion & sparkle animations:**
- Wrong tap: 8 red/orange fragments shoot outward 35-50px, fade + scale over 400ms; screen shakes ±3px twice; red edge flash; `-❤️` popup; `sfx_skeet_explosion.wav`
- Correct tap: 8 gold/white sparkle fragments; green edge flash; `+N 🎯` popup; `sfx_combo.wav`

**New audio:**
- `sfx_skeet_launch.wav` — descending whoosh, 0.25s; plays on every skeet spawn
- `sfx_skeet_explosion.wav` — noisy burst, 0.35s; plays on wrong tap
- Both generated by `scripts/generate_skeet_sounds.py`; both registered in `assets/audio/`

**AudioService:** `playSkeetLaunch()`, `playSkeetExplosion()` added  
**SettingsService:** `skeetUsePhonetic` getter + `setSkeetUsePhonetic(bool)` + load/save  
**ArcadeCountdownWidget:** optional `extraContent` widget parameter added (backward-compatible)

## Architecture Notes

- **Singleton services** with factory constructor
- **JSON-driven lessons** — every lesson is a self-contained JSON asset (`assets/lessons/`)
- **SharedPreferences** for all persistence (progress, settings, review queue)
- **Stage chains** define both unlock order and color gradient position
- **`_lessonFillColor(id)`** — looks up chain position → lerps color anchors
- **Stage 0 ID space** — uses 101-105 to avoid collision with Stage 1/2/3 IDs 1-50
